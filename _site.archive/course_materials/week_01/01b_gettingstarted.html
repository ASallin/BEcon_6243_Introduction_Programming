On files, paths, and shortcuts

With the terminal, you access and work with files. A file is a container of data (remember from Data Handling: 0s and 1s).

On files, paths, and shortcuts

To access files, their location must be given with paths. Paths describe where a file is located.

Absolute path:
  • starts from the root of the file system
  • example: C:/Users/Bilbo/Documents/TheRedBookofWestmarch.txt
Relative path:
  • starts from your current working directory
  • example: Documents/TheRedBookofWestmarch.txt

Main terminal commands

As said, when you first open Terminal you typically are placed in your home directory.

Main commands:
  • Check your current directory: pwd (print working directory)
  • List the files in the current directory: ls (macOS, Linux) or dir (Windows)
  • Move into and out of the directory: cd dir or cd .. (change directory)
  • Running cd without arguments takes you to ~
Tips:
  • Use Tab for auto-completion
  • Use arrow keys to browse command history
  • Use clear (macOS/Linux) or Ctrl+L to clear the screen

On IDEs

What is an IDE?

An IDE (Integrated Development Environment) is a tool designed to help you write, run, and manage code efficiently.

Compared to a basic text editor, an IDE typically provides:

  • syntax highlighting
  • code completion and suggestions
  • error detection
  • integrated terminal
  • debugging tools
  • version control integration (e.g.Â Git)

These features reduce errors and make programming more structured and productive.

Examples: VSCode, Pycharm, RStudio, Posit

Environments in Python and R

A first striking difference (and a source of lots of sweat) between R and Python is dependency management.

In R:

  • strong(er) standards (CRAN)
  • more backward compatibility

In Python:

  • Python relies heavily on project-specific environments
  • Managing dependencies is essential
  • âWild Westâ: many packages, many versions
  • environments are critical to avoid conflicts

Virtual environment

A virtual environment is an isolated Python setup for a project.

  • install project-specific packages and python version for a specific project without affecting other projects
  • use different versions across projects
  • avoid conflicts and broken setups, minimizes the risk of bad surprises

A project is simply a folder that contains your code, its configuration, and its dependencies.

Virtual Environment Manager: uv

uv is a modern, fast Python package manager that simplifies working with environment:

  • manages virtual environments automatically
  • installs packages very fast
  • works consistently across systems
  • uses modern project standards

(Replaces pip, pip-tools, and virtualenv.)

Virtual Environment Manager: uv

With uv, each project has:

  • its own environment
  • a clear list of dependencies
  • reproducible setup

Getting started using uv

# Initialize a new project with its own environment
uv init my_project
cd my_project

# Add dependencies (creates/updates pyproject.toml)
uv add rich pandas

# Run code (automatically uses the project's venv)
uv run main.py

Project based on the pyproject.toml.

In Python, indentation defines the structure of your code.

Blocks of code are created by indentation, not by curly braces {} like in many other languages.
